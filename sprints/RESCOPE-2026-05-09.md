## Layer model (the distinction)

This repo holds two layers. The v2 plan must respect them.

| Layer | What it is | Where it lives | Maintained by |
|---|---|---|---|
| **Agent-11** (framework) | Generic dev squad used to build SEO-Agent | `.claude/agents/`, `.claude/CLAUDE.md`, `.claude/commands/{coord,architect,bootstrap,foundations,plan,…}.md`, `/missions/mission-*.md`, `/templates/` (non-seo), `/field-manual/`, `/gates/`, `/schemas/`, `/stack-profiles/` | Upstream Agent-11; refresh via `bash install.sh --upgrade` |
| **SEO-Agent** (product) | The SEO suite we build here and deploy to do SEO work elsewhere | `/agents/seo-*.md` (7 agents), `.claude/missions/{site-audit,content-gap,technical-fix}.md`, `/missions/ai-search-optimize.md`, `/templates/seo-*-template.md`, `.claude/commands/{seo-commands,track,tracking-commands}.md` | Us, in this repo |

The v6.1.1 upgrade only touched the framework layer. The product layer is untouched.

**Implication**: stop talking about deleting framework files. Treat the framework as a dependency. SEO-Agent sprints work on the product layer only.

## Decisions (locked 2026-05-09)

1. **Mission file location**: move `ai-search-optimize.md` from `/missions/` into `.claude/missions/`. All four SEO missions sit together. Captured as a Sprint 1 task.
2. **NLP routing**: dropped entirely. Sprint 4 stays cancelled. No standalone `/seo` parser. Users dispatch via positional args (`/coord site-audit lite freecalchub.com/calculators`).
3. **Order**: Sprint 2 runs first (cleanest, zero deps, gets v5 prose out of project-root `CLAUDE.md` before Sprint 3 touches the agents). Sprint 1 can run in parallel with Sprint 2.
4. **Gate**: kept, but **conditional on the sprint exercising the runtime path**. Sprints that change config or prose only (1, 2) close on lighter evidence (diff + smoke dispatch). Sprints that change runtime behaviour (3, 5, 6, 7) keep the full freecalchub.com audit before/after.
